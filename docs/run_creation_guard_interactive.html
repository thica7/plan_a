Competiscope v2 任务防重复与产品交互说明

本文件说明当前项目做什么、输入什么会得到什么输出，以及本次为 Start run 增加的前端校验与后端活跃任务防重复策略。

当前项目是干什么的

Competiscope v2 是一个竞争情报分析工作台。它把主题、竞品、分析维度、 证据采集、知识库检索、Agent 编排、人工审核、报告生成和质量评估串成一个 run。

FastAPI 后端 React/Vite 控制台 RAG/KB LangGraph/Temporal QA + Revision

典型业务价值

给定一个市场主题或竞品列表，系统自动组织证据，输出可引用、可审计、 可重跑修复的竞品对比报告，而不是只生成一段不可追踪的普通文本。

输入什么，得到什么

输入 示例 输出
分析主题 Agentic AI IDE competitive analysis 创建一个 run，生成计划、事件流、分析报告和质量指标。
竞品模式 Auto-discover 或 Cursor, Copilot, Windsurf 自动发现或按指定名单生成竞品知识与对比矩阵。
分析维度 pricing, feature, persona, security 按维度采集证据、提取事实、生成矩阵单元格。
执行模式 Demo 或 Real API Demo 使用本地/模拟路径；Real API 调用真实 LLM 和搜索配置。
人工审核选项 HITL enabled 在 planner/QA 节点暂停，等待人工 accept、modify、redo 或 force pass。

Start run 防重复方案

前端校验

提交前检查主题长度、维度数量、手动竞品数量、Real API 配置和 workspace quota。

幂等 Key

前端仍发送 idempotency_key，同 key 重试会复用同一个 run。

任务指纹

后端规范化 workspace、topic、competitors、dimensions、scenario、mode 后生成指纹。

活跃查重

5 分钟内已有相同指纹且状态为 queued/running/interrupted 时，直接返回已有 run。

完成后可重跑

completed、completed_with_blockers、failed 不会被拦截，允许用户重新分析。

仍建议开发

  • 用户/IP 级别的 POST /runs 限流，防止脚本换 payload 大量创建任务。
  • 正式 i18n key 管理，目前主要还是硬编码文案。
  • Real API 错误恢复体验，包括可视化重试、原因分层和成本预估。
  • 租户、权限、审批流的前端产品化。
  • 知识库治理仪表盘，将重复、过期、低质量证据以任务形式呈现。

本次已落地

  • 前端创建任务前校验。
  • 后端 active-run fingerprint 防重复。
  • 不同 idempotency key 但同任务内容时复用活跃 run。
  • 完成后的 run 允许重新创建。
  • 针对性后端单元测试和前端构建验证。

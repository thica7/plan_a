# Clean Research Pipeline Refactor Summary

Date: 2026-06-06

## Goal

This refactor turns evidence collection from a collector-centered bundle of
heuristics into a typed research pipeline:

```text
Discover -> Capture -> Extract -> Evidence items -> Evaluate -> Repair
```

The collector still owns LangGraph orchestration, tracing, and run state. The
research package now owns source candidate identity, capture normalization,
field extraction, quality gaps, and repair task contracts.

## Module Boundaries

| Module | Owns | Does not own |
| --- | --- | --- |
| `packages.research.models` | Typed contracts and stable IDs | Network I/O |
| `packages.research.discovery` | Candidate providers, search query planning, ranking, URL dedupe | Page fetching |
| `packages.research.capture` | Fetch-result normalization into `CapturedPage` | Business evidence rules |
| `packages.research.extraction` | Pricing, feature-slot, persona field extraction | Candidate discovery |
| `packages.research.evidence` | RawSource admission and lineage | Search planning |
| `packages.research.evaluation` | Structured `QualityGap` creation | Fetching or report writing |
| `packages.research.repair` | `QualityGap` to `RepairTask` conversion | Direct mutation of run state |
| `packages.research.pipeline` | Single-branch pipeline composition | LangGraph/Temporal orchestration |

## Core Contracts

`ResearchBrief`

- One competitor x one dimension x one run.
- Includes topic, homepage hint, execution mode, source/fetch limits, required
  fields, and repair gap context.

`SourceCandidate`

- A candidate URL before fetch.
- Origin is one of `trusted_registry`, `perplexity`, `web_search`,
  `homepage_derived`, `llm_fallback`, or `manual`.
- Stable ID is derived from origin, competitor, dimension, URL, and rank.

`CapturedPage`

- Normalized fetch output.
- Accepts both new `EvidenceFetchResult` and legacy `FetchPageResult` shapes.
- Represents failed or missing fetches explicitly instead of returning `None`.

`ExtractionResult`

- Structured fields extracted from one captured page.
- Current extractors: pricing model, feature slots, persona schema.
- Supports `not_applicable_reason` for cases such as open-weight Llama pricing.

`QualityGap`

- Typed evaluation output with severity, target field, action, and acceptance
  rule.
- This is the bridge from QA-warning/GAP-driven plans to deterministic repair.

`RepairTask`

- Targeted discovery/capture/extraction instruction derived from a gap.
- Carries query hints and acceptance rules for the next repair pass.

## ID And Reference Rules

- Candidate IDs use `source-candidate-*`.
- Capture IDs use `captured-page-*`.
- Extraction IDs use `extraction-*`.
- Evidence item IDs use `evidence-item-*`.
- Quality gaps use `quality-gap-*`.
- Repair tasks use `repair-task-*`.
- `RawSource` IDs remain generated by `compute_raw_source_id()` through
  `raw_source_from_capture()` so report/source references stay compatible with
  the existing run and enterprise projection layers.

The important rule is: discovery/capture/extraction IDs describe research
pipeline lineage; `RawSource.id` remains the canonical citation ID for reports.

## Current Collector Integration

The collector now uses the research package for:

- Trusted and homepage official source candidates.
- Search/Perplexity result candidate normalization and dedupe.
- Page capture normalization.
- RawSource construction from captured pages.
- Shared evidence quality rules.

The collector still owns:

- LangGraph node execution.
- Trace and local tool events.
- Run-state mutation.
- Survey/interview synthetic records.
- Existing QA/redo graph control.

## QA-Warning And GAP-Driven Repair

The repair path is now explicit:

```text
ExtractionResult[]
  -> quality_gaps_from_extractions()
  -> QualityGap[]
  -> repair_tasks_from_gaps()
  -> RepairTask[]
  -> build_search_queries(brief, repair_tasks=...)
```

This keeps repair behavior data-driven. It avoids scattering natural-language
QA warning parsing through the collector.

## Follow-Up Integration Points

- Feed final QA/release-gate blockers into `QualityGap` instead of ad hoc text
  warnings.
- Let automatic redo consume `RepairTask` objects directly.
- Add field-level admission for pricing tiers, persona segments, and feature
  slots after analyst normalization.
- Add pipeline-level cache and source saturation metrics when repeated fetches
  become the next bottleneck.
